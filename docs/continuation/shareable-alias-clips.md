# Shareable Alias Clips — Status

## Goal
Let users share generated audio for player-name-placeholder dialog with each other.
Approach: alongside the personal generation, optionally produce two extra "alias" generations
per placeholder clip, using a localized generic noun ("Adventurer" / "Abenteurer(in)" /
"Aventurier(ière)" / "冒険者") as the substituted player name. These variants are
gender-specific and detached from any particular player, so any user can play back the male
or female alias variant matching their character.

## Done

### Schema v13
- `voice_clip_generations.alias_gender` column (0=real player, 1=male alias, 2=female alias).
- Recreated unique index as `(voice_clip_id, player_content_id, alias_gender)` so the personal
  row and both alias variants coexist for the same clip.
- Migration in `DatabaseService.RunSchemaMigrations()`. Idempotent on fresh installs.

### DB API
- `IDatabaseService.LogVoiceClipGeneration / GetVoiceClipGeneration / DeleteVoiceClipGeneration`
  all take an optional `aliasGender` (default 0). Alias rows always pass `playerContentId=0`.

### Generation pipeline (`VoiceClipManagerService`)
- New `BuildVoiceMessageWithAlias(clip, isMale)` produces a `VoiceMessage` with the placeholder
  substituted by `TalkTextHelper.GetPlayerAlias(clip.Language, isMale)` and `isMale` carried
  through for `-IfGender-` resolution.
- `GenerateAliasVariant(clip, isMale)` — backend call → write file → log generation row with
  the chosen `aliasGender`. Best-effort: warning on failure, never bubbles up to caller.
- `GenerateForVoiceClip` auto-runs both alias variants after the main generation succeeds when
  `Configuration.AutoGenerateShareableAliases` is true and `voiceClip.HasPlayerPlaceholder`.

### Localization helper
- `TalkTextHelper.GetPlayerAlias(ClientLanguage, bool isMale)` → DE/FR/JP/EN strings.
  EN/JP have no gender variants — both isMale=true/false return the same string. The DB still
  carries two rows for those languages so import/export logic can stay gender-keyed.

### Configuration
- `Configuration.AutoGenerateShareableAliases` (bool, default false).
- UI: checkbox in `NativeConfigWindow.BuildSaveLoadPanel` ("Auto-generate shareable alias
  variants for player-name dialog"). Loc keys for DE/FR/JP added.

### Tests
- `VoiceClipGenerationTests` — schema column present, alias variants coexist with personal,
  Get/Delete with `aliasGender`, upsert by composite key.
- `TalkTextHelperTests.GetPlayerAlias_ReturnsLocalizedGenderedNoun` — Theory across 8
  (lang, isMale) combinations.
- `TalkTextHelperTests.SubstitutePlaceholders_AcceptsAliasAsPlayerName` — alias passes through
  the same substitution code path as the real name.

## Open — Export / Import

The only remaining TODO. Needs a design pass before code:

- **Export format**: zip of (alias `.wav` files + a manifest JSON) or per-NPC folders mirroring
  the on-disk layout? Manifest must carry enough to re-link on import:
  `(character name + race + gender + language, original_text, alias_gender, voice_key?)`.
  Voice key is probably **not** stable across installs — likely re-generated by the importer
  using their own voice assignments rather than carried over.
- **Scope of one export**: single character / per-language / everything? Bulk by default.
- **Import behaviour**: match existing characters by `(name, gender, race, language)` via
  `FindCharacter`. Voice clips matched by `(character_id, npc_base_id, original_text)` like
  the existing composite index. Insert generation rows with `alias_gender=1|2` and
  `player_content_id=0`.
- **Conflict policy**: skip / overwrite / per-row prompt? Suggest skip-if-exists by default
  with an "overwrite" toggle in the import UI.
- **UI placement**: probably two buttons in `NativeConfigWindow` Save/Loading panel right
  below the alias-auto-generation checkbox, or inside `NativeVoiceClipManagerWindow`. To be
  decided.
